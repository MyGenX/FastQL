---
title: Apollo Federation v2
description: Federation directives, entity resolution, and the _service SDL.
---

`fastql.federation` adds Apollo Federation v2 support: the federation directives as
applied-directive metadata, a federated-SDL printer with the `@link` import header,
and the `_service` / `_entities` machinery with per-type reference resolvers.

```python
from fastql import Field, Query, Type
from fastql.federation import (
    key, shareable, reference_resolver, print_federated_schema,
)


@Type(directives=[key("id"), shareable()])
class Product:
    id: str
    name: str


@reference_resolver(Product)
def resolve_product(representation: dict) -> Product:
    return PRODUCTS[representation["id"]]


sdl = print_federated_schema(schema)   # includes the @link(url: ...) header
```

- Directive helpers: `key`, `external`, `shareable`, `requires`, `provides`,
  `inaccessible`, `override`, `tag` — applied via `@Type(directives=[...])` or
  `@Field(directives=[...])`.
- The federated `Schema` exposes `_service { sdl }`, the `_Any` / `_Entity` /
  `_Service` types, and `_entities(representations:)`.
- `@reference_resolver(Type)` (or `register_reference`) registers how to rehydrate an
  entity from its key fields; `_entities` returns resolved entities in request order.
